JellyOS is a terminal AI trading agent for blockchain analytics, prediction markets, and multi-chain portfolio management. JellyOS - terminal UI, model routing, agent loop, session management, and context compaction. JellyOS provides the domain: blockchain tools, trading logic, encrypted vault, live data feeds, and skills.

## Features

- **Pi-powered terminal** ‚ÄĒ full TUI with streaming AI responses, session branching, context compaction, model switching
- **Any model** ‚ÄĒ configure any OpenRouter, OpenAI, Anthropic, Gemini, or local model via Pi's model registry
- **22 domain tools** ‚ÄĒ market data, funding rates, fear/greed, DeFi TVL, Polymarket, wallet balances, whale scanning, vault, trading, feeds, signals, prediction
- **16-chain support** ‚ÄĒ Ethereum, BSC, Arbitrum, Base, Polygon, Avalanche, Optimism, Scroll, Linea, ZkSync, Mantle, Blast, Gnosis, Celo, Solana, and more
- **Live data feeds** ‚ÄĒ always-on sources: prices, news, fear/greed, DeFi TVL, funding rates, on-chain whale data
- **Signal engine** ‚ÄĒ auto-generates directional trading signals by cross-referencing multiple feed sources
- **Encrypted vault** ‚ÄĒ AES-256-GCM profit vault with scrypt KDF; sweep realized profits in after every trade
- **Multi-chain wallets** ‚ÄĒ EVM (secp256k1), Solana (ed25519), Cosmos (ed25519) keypairs; sign without exposing private keys
- **Skills** ‚ÄĒ Pi skill system for trading, analysis, and vault discipline guides built into context
- **Jelly theme** ‚ÄĒ custom cyan/purple dark theme shipped with the package

## Requirements

- **Node.js 20+** (LTS recommended)
- **npm 9+**
- An API key for an AI model gateway (`OPENROUTER_API_KEY`, or configure any Pi-supported provider)
- Optional: an RPC node API key for on-chain data (`ALCHEMY_KEY`)

## Installation

```bash
# Clone
git clone https://github.com/jelly-chain/jellyOS.git
cd JellyOS

# One-command setup (Linux / macOS)
bash setup.sh
```

`setup.sh` will:
1. Check your Node.js version (20+ required)
3. Run `npm install`
4. Launch the interactive setup wizard (`node bin/jellyos setup`) which prompts for your API keys and writes them to `~/.jelly/.env`

**Windows:**
```powershell
powershell -ExecutionPolicy Bypass -File setup.ps1
```

**Manual setup:**
```bash
npm install
npm run build
node bin/jellyos setup
```

## How the Agent Works

Every message you type goes through a pipeline:

```
Your input
    ‚Ēā
    ‚Ėľ
SwarmRouter.shouldUseSwarm()
    ‚Ēā
    ‚Ēú‚ĒÄ NO  ‚Üí streamCompletion()
    ‚Ēā            ‚Ēā
    ‚Ēā            ‚Ēú‚ĒÄ ModelRouter picks MODEL_1 (with failover to MODEL_2..5)
    ‚Ēā            ‚Ēú‚ĒÄ Sends messages + tool definitions to the model
    ‚Ēā            ‚Ēú‚ĒÄ Streams response tokens to your terminal
    ‚Ēā            ‚ĒĒ‚ĒÄ If model calls a tool ‚Üí executes tool ‚Üí continues conversation
    ‚Ēā
    ‚ĒĒ‚ĒÄ YES (turbo/max + complex prompt)
                 ‚Ēā
                 ‚Ėľ
             runSwarm()
                 ‚Ēā
                 ‚Ēú‚ĒÄ SwarmRouter.splitIntoTasks() ‚ÄĒ breaks prompt into sub-questions
                 ‚Ēā     (groups of 3, up to 5 worker agents + 1 reviewer)
                 ‚Ēā
                 ‚Ēú‚ĒÄ SubAgentManager.runParallel() ‚ÄĒ fires all sub-agents simultaneously
                 ‚Ēā     each sub-agent = separate HTTP call on its own model
                 ‚Ēā
                 ‚Ēú‚ĒÄ SubAgentManager.synthesize() ‚ÄĒ reviewer agent reads all results
                 ‚Ēā     and writes a unified answer
                 ‚Ēā
                 ‚ĒĒ‚ĒÄ Final response streamed to terminal
```

### Model failover

Every request attempts models in order `MODEL_1 ‚Üí MODEL_2 ‚Üí MODEL_3 ‚Üí MODEL_4 ‚Üí MODEL_5`. If a model returns a rate-limit error (429) or a server error (5xx), the engine immediately retries the request on the next model without dropping back to the user. Only if all five fail does an error appear in the terminal.

### AutoVault

A background timer checks the signal engine's estimated net P&L every 60 seconds. If it exceeds the `AUTO_VAULT_THRESHOLD` and the vault is unlocked, it automatically sweeps that amount into your encrypted vault. The vault then locks until you run `/unlock <passphrase>` again.

## How Dispatch Works Per Level

### eco (30%) ‚ÄĒ single model, zero overhead
```
User: "check BSC balance for 0x..."
‚Üí Main agent uses MODEL_4 (lowest cost)
‚Üí Calls get_balance tool directly
‚Üí Done. No sub-agent.
```

### normal (50%) ‚ÄĒ main + optional 1 sub-agent
```
User: "analyze ETH"
‚Üí Main agent uses MODEL_1
‚Üí Dispatches 1 sub-agent on MODEL_2 for quick chain data
‚Üí Synthesizes response
```

### turbo (70%) ‚ÄĒ parallel sub-agents
```
User: "analyze ETH and check if I should trade"
‚Üí Main agent uses MODEL_1
‚Üí Sub-agent A (MODEL_3): sentiment + prediction
‚Üí Sub-agent B (MODEL_2): positions + risk
‚Üí Both run in parallel
‚Üí Main synthesizes ‚Üí "ETH looks bullish, you have no open position. Buy?"
```

### max (100%) ‚ÄĒ full swarm
```
User: "scan BSC for whale activity, check my portfolio, predict top movers"
‚Üí Main agent uses MODEL_1
‚Üí Sub-agent A (MODEL_3): scan 500 blocks for whale transactions
‚Üí Sub-agent B (MODEL_5): portfolio analysis + P&L
‚Üí Sub-agent C (MODEL_2): predict top movers using signal engine
‚Üí All run in parallel
‚Üí Reviewer (MODEL_1) synthesizes all results
‚Üí "3 whale movements detected in last 50 blocks, portfolio is +4.2%, ETH/SOL flagged as top movers"
```


## Security

- **Private keys** are stored in `wallets/` at the project root ‚ÄĒ this directory is in `.gitignore`. Never commit it and keep the directory private.
- **Vault** is encrypted with AES-256-GCM. The key is derived from your passphrase using scrypt (memory-hard KDF) with a random per-vault salt. The same salt is stored in the vault file so the key can be reproduced deterministically from your passphrase ‚ÄĒ but the key itself is never persisted.
- **Private keys never leave the process** ‚ÄĒ signing happens in memory and only the resulting signature is returned.
- **API keys** are read from `~/.jellyos/.env` at startup and never stored anywhere else or logged.
- **Auto-lock** ‚ÄĒ vault is locked automatically on `/panic` and on process exit.
